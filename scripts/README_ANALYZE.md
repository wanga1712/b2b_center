# Инструкция по анализу базы данных tender_monitor

## Шаг 1: Добавление настроек в .env файл

Добавьте следующие строки в ваш `.env` файл (в корне проекта):

```env
# База данных tender_monitor (торги)
TENDER_MONITOR_DB_HOST=localhost
TENDER_MONITOR_DB_DATABASE=tender_monitor
TENDER_MONITOR_DB_USER=postgres
TENDER_MONITOR_DB_PASSWORD=0IFz3_
TENDER_MONITOR_DB_PORT=5432
```

## Шаг 2: Запуск скрипта анализа

Запустите скрипт из корневой директории проекта:

```bash
python scripts/analyze_tender_db.py
```

Или из виртуального окружения:

```bash
venv\Scripts\python.exe scripts\analyze_tender_db.py
```

## Что делает скрипт

Скрипт анализирует базу данных `tender_monitor` и выводит:

1. **Список всех таблиц** - все таблицы в базе данных
2. **Структуру каждой таблицы**:
   - Названия колонок
   - Типы данных
   - Ограничения (NULL/NOT NULL)
   - Значения по умолчанию
3. **Первичные ключи** - какие поля являются первичными ключами
4. **Внешние ключи** - связи между таблицами
5. **Индексы** - какие индексы созданы для ускорения поиска
6. **Количество строк** - сколько записей в каждой таблице
7. **Примеры данных** - первые 3 записи из каждой таблицы

## Результат

Скрипт выводит информацию в консоль и сохраняет детальный отчет в JSON файл:
- `tender_db_analysis_YYYYMMDD_HHMMSS.json`

Этот файл содержит всю информацию о структуре базы данных в структурированном виде.

## Использование результатов

После получения результатов анализа, предоставьте их для дальнейшей настройки модуля "Торги" в приложении.
